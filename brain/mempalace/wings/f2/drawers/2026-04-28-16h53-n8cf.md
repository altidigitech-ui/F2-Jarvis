### **Le bug principal : `action-execute-batch.ts` ligne 85**

La route que l'UI appelle quand tu cliques "Valider" est `/action/execute-batch`. Elle traite **deux chemins différents** selon le type d'action :

- `create_file` → passe par `await executeAction(action.id)` ✅
- **tout le reste** (`log_engagement`, `batch_cold`, `mark_published`, etc.) → bypass complet d'`executeAction` ❌

Pour le chemin "tout le reste", voici ce qui se passe ligne 85 :

```typescript
const { path } = resolveFilePath(
  action.action_type,
  action.jarvis_conversations.persona,  // ← "romain" en dur
  action.params
);
```

Le deuxième argument est `action.jarvis_conversations.persona`. Dans la DB, la conversation F2 a `persona = "romain"` (c'est Romain qui gère @foundrytwo). La logique `_persona_prefix` qui corrige ça existe dans `executeAction` — mais `action-execute-batch.ts` **ne l'appelle jamais pour les actions normales**. Il utilise directement le `persona` brut de la DB.

Résultat : path = `romain/engagement/engagement-log.md`. Toujours.
